이런 저런 이유로 기존 블로그에 청구 요금이 붙기 시작했다.

수익을 내려고 만든게 아니다보니 돈이 아깝기도 했고

기존 블로그는 실용성이 전혀 없어서 이참에 새로 하나 만들자는 생각이 스멀스멀 올라오다보니 자꾸 눈에 mdx가 밟혔다.. ㅋㅋ

## 프로젝트 구조 설계하기

코딩하다가 깜빡깜빡할 때 찾아보는 용도로 쓰는게 목표라서 거의 모든 블로그에 있는 목차(table of contents)나 네비게이션, 시리즈, 태그는 내가 쓰기엔 그다지 필요 없었고
댓글기능, 연관 포스트, 시리즈 등 챕터를 나누는 것도 만들어 놓고 써본적이 없었다.

생각해보니 DB도 S3같은 스토리지도 필요 없더이다.

단축키, 포매팅, 문법 검사, 자동완성 다 되는 vscode놔두고 웹에서 힘들게 md파일 편집할 이유도 없는 것 같다.

목차, 검색도 파일 기반으로 하면 되는 것이고 정적 파일도 github 저장소 하나면 평생 무료로 써도 충분하다는 생각으로 시작했다.

라즈베리파이를 하나 사서 렌선에 꼽고 개발을 착수했다.

```bash
src
├── app
│   ├── posts
│   │   ├── @contents
│   │   │   ├── example
│   │   │   │   ├── architecture
│   │   │   │   │   ├── [id]
│   │   │   │   │   │     ├── page.tsx
│   │   │   │   │   │     ├── 1. post.mdx
│   │   │   │   │   │     ├── 2. post.mdx
```

이런 구조로 만들었고 목차, 검색 기능 모두 서버에서 파일 구조를 내려서 읽어주는 걸로 해결했다.
